# Документация сервиса "Табло"

## Краткое описание

Сервис "Табло" принимает на себя следующие обязательства:
1. Создание сущности Рейс (Flight) на основе присланного самолета (Airplane) сервисом "Самолет";
2. Уведомление сервисов "Пассажир" и "Касса" о создании рейса;
3. Уведомление сервисов "Пассажир" и "Касса" о начале регистрации (на основе уведомление от сервиса "Управление наземным обслуживанием");
4. Ожидание заранее установленного промежутка модельного времени после начала регистрации;
5. Уведомление сервисов "Пассажир" и "Касса" о конце регистрации;

## Архитектура и зависимости

Решение состоит из 5 проектов:
1. DepartureBoard.Domain
Данный проект определяет основные сущности сервиса -- Самолет (Airplane) и Рейс (Flight), а также набор констант для работы со временем.
2. DepartureBoard.Application
Данный проект содержит юз-кейсы, службу времени и порты (абстракции) для работы с базами данных и другими сервисами.
3. DepartureBoard.Infrastructure
Данный проект содержит адаптеры (имплементации) портов из вышестоящего проекта. Также содержит DbContext приложения и все его миграции.
Зависимости:
NuGet:
- Npgsql.EntityFrameworkCore.PostgreSQL (9.0.4)
4. DepartureBoard.Api
Данный проект содержит API сервиса, контроллеры, middleware и различные опции.
Зависимости:
NuGet:
- Microsoft.AspNetCore.OpenApi (9.0.0)
- Microsoft.EntityFrameworkCore.Design (9.0.3)
- Swashbuckle.AspNetCore (7.3.1)
5. DepartureBoard.Desktop
Данный проект реализует простое WPF-приложение для управления модельным временем.
